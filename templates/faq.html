Foire Aux Questions

Q: Comment avez-vous décomposé le projet ?
A: Le projet a été décomposé en cinq phases principales, suivant les étapes clés recommandées :
  • Phase 1 - Sélection et Test de l'API (1 semaine) :
    • Recherche d'APIs respectant les critères (REST, JSON, sans quota)
    • Tests des endpoints avec Postman
    • Documentation des limites et des réponses de l'API
    • Validation du choix avec l'encadrant
  • Phase 2 - Design et Maquettage (1 semaine) :
    • Création des wireframes pour toutes les pages requises
    • Design de l'interface utilisateur selon les contraintes techniques
    • Validation de l'arborescence du site
  • Phase 3 - Développement (2 semaines) :
    • Implémentation des fonctionnalités obligatoires (FT1 à FT4)
    • Développement du système de recherche et filtrage
    • Création du système de pagination
    • Mise en place du système de favoris avec persistance
  • Phase 4 - Tests et Débogage (3 jours) :
    • Tests de toutes les fonctionnalités
    • Vérification de la gestion des erreurs
    • Tests de compatibilité navigateur
    • Correction des bugs identifiés
  • Phase 5 - Documentation et Finalisation (4 jours) :
    • Rédaction du README
    • Documentation des routes
    • Création de la page à propos
    • Préparation du livrable final
  • Phase 6 - Résolution des Problèmes et Optimisation (1 semaine) :
    • Correction des bugs majeurs de l'interface utilisateur
    • Résolution des problèmes de performance avec l'API
    • Amélioration du système de gestion d'erreurs
    • Optimisation du cache et des temps de réponse
    • Correction des problèmes de compatibilité navigateur
    • Amélioration de la validation des données
    • Résolution des problèmes de persistance des favoris
    • Optimisation des requêtes de recherche et filtrage
Q: Comment avez-vous géré votre temps ?
A: La gestion du temps a été structurée selon plusieurs axes :
  • Planification :
    • Points d'étape hebdomadaires
    • Réserve de 3 jours pour les imprévus
    • Définition des priorités en début de semaine
    • Alternance entre périodes de travail et pauses
  • Priorisation :
    • Focus initial sur les fonctionnalités obligatoires
    • Développement par ordre de dépendance technique
    • Tests en continu pour éviter l'accumulation de bugs
    • Fonctionnalités bonus uniquement après validation des essentielles
  • Organisation quotidienne :
    • 2 à 4 heures de développement environ chaque journée
    • Tests et corrections l'après-midi
    • Documentation en fin de journée
    • Revue de code quotidienne
  • Gestion des risques :
    • Identification des points critiques dès le début
    • Solutions de repli préparées pour les fonctionnalités complexes
    • Veille sur les limitations de l'API
    • Suivi régulier de l'avancement
Q: Quelle stratégie avez-vous adoptée pour la documentation ?
A: Notre approche de la documentation s'est faite sur plusieurs niveaux :
  • Documentation technique :
    • README détaillé à la racine du projet
    • Documentation des routes et endpoints
    • Guide d'installation et de configuration
    • Description des dépendances et prérequis
  • Documentation utilisateur :
    • Guide d'utilisation des fonctionnalités
    • Page à propos complète
    • FAQ détaillée
    • Captures d'écran des principales fonctionnalités
  • Documentation du code :
    • Commentaires pertinents dans le code
    • Nomenclature cohérente
    • Structure des dossiers expliquée
    • Documentation des fonctions principales
  • Documentation du projet :
    • Suivi des décisions techniques
    • Journal des difficultés rencontrées
    • Solutions mises en place
    • Pistes d'amélioration futures
Q: Quels ont été les principaux défis techniques ?
A: Le projet a présenté plusieurs défis techniques majeurs :
  • Gestion des contraintes :
    • Utilisation exclusive de Go, HTML,JS et CSS
    • Pas de frameworks externes
    • Gestion des erreurs robuste
  • Implémentation des fonctionnalités :
    • Système de recherche sur multiple critères
    • Filtres cumulables
    • Pagination efficace
    • Persistance des favoris
  • Performance :
    • Optimisation des requêtes API
    • Gestion du cache
    • Temps de réponse optimisés
    • Charge serveur minimisée
  • Expérience utilisateur :
    • Interface intuitive
    • Feedback utilisateur clair
    • Navigation fluide
    • Gestion des erreurs user-friendly
